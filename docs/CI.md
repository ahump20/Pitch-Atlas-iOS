# CI / Release

## GitHub Actions

`.github/workflows/ios.yml` is the PR/main check. It installs XcodeGen, generates
`PitchAtlas.xcodeproj`, builds the app with `xcodebuild`, then runs unit tests on
an available iPhone simulator. Do not replace it with `swift build` / `swift test`;
this app is not a SwiftPM executable package.

`scripts/build.sh test` must fail when tests fail. The wrapper uses `pipefail`
and only pipes through `xcpretty` when `xcpretty` exists.

## Xcode Cloud

`PitchAtlas.xcodeproj` is generated by **XcodeGen** from `project.yml` and is not
hand-edited. Xcode Cloud runs `ci_scripts/ci_post_clone.sh` after clone; that
script installs XcodeGen when needed and runs `xcodegen generate`.

The bundled content (`Resources/Content/*.json`) **is** committed, so CI needs no
web repo and no Node. Refreshing content from the web source is a deliberate,
local pre-release step (`cd tools/generate-content && npm i && npm run generate`),
run and committed before tagging a release. CI does not do this per build.

## Workflows to create (in Xcode → Cloud, or App Store Connect)

1. **CI**: on push to `main` and PRs, Build + Test (the `PitchAtlas` scheme,
   iOS Simulator). No archive. Fast feedback.
2. **Release**: on a release branch or tag, Build, Test, Archive, then TestFlight.
   This is the one that uploads.

Both run `ci_post_clone.sh` automatically (Xcode Cloud convention: any executable
in `ci_scripts/` named `ci_post_clone.sh` runs after clone).

## Release conventions

- **Build numbers are monotonic.** Xcode Cloud increments the build number above
  whatever is in `project.yml`'s `CURRENT_PROJECT_VERSION` at archive time; never
  hand-set a number below the last TestFlight build.
- **`action_required` means a compile error**, not a manual approval gate. Read
  the build log's error count and fix the code.
- **Same-day merge-back:** a release branch merges back to `main` the day it
  archives, so `main` never drifts behind a shipped build.

## Local build / test

```bash
xcodegen generate
xcodebuild -project PitchAtlas.xcodeproj -scheme PitchAtlas \
  -destination 'platform=iOS Simulator,name=iPhone 17 Pro' test
```
