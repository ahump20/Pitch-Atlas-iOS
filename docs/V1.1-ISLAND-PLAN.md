# v1.1 — the interactive WebGL specimen island (deferred)

v1 ships the **native** specimen (`SeamBall`) — the seam drawn from the real
figure-eight equation, oriented to the spin axis, with the physics-derived break
indicator and the gyroscope foil. That already satisfies the specimen role and
keeps physics from drifting (it uses the same equation as the web).

The **interactive 3D** version (drag-to-spin, live spin-axis vector, the
3D→2D dissolve) is deferred to v1.1 deliberately, because it is the one piece
that:

- needs **on-device** verification (WebGL inside `WKWebView` under a custom URL
  scheme, offline), which the simulator can't fully prove; and
- adds review surface (a web view in a content app) to a *first* submission.

Keeping it out makes the first review cleaner (no web view = no remote-code or
offline questions at all) without losing the specimen experience.

## The plan when we add it

1. **Minimal Three.js entry, not the full site.** The web's `BallScene`
   (`@react-three/fiber` + `drei` + `three`) is cleanly parameterized by a
   `PitchAtlasEntry`. Build a dedicated `island/index.html` + `island/main.tsx`
   that mounts **only** `<BallScene entry autoSpin />`, reads the pitch slug from
   the URL hash, transparent background. A full prerendered route would pull in
   the whole page (a wrapper → 4.2 risk) and uses `createBrowserRouter` absolute
   paths that 404 under a custom scheme — so a dedicated minimal entry is the
   only correct design.
2. **Separate Vite build** with `base: './'` (relative assets) → its own `dist`.
   Strip the embeds (`PitchingNinjaTweet`, `StreamableFacade`, the tweet JSON) —
   the minimal entry won't import them, but assert it.
3. **Sanitize + prove offline:** a network capture of the bundle in a `WKWebView`
   shows zero off-device fetch and zero remote `script-src`.
4. **`PitchAtlasSchemeHandler`** (`WKURLSchemeHandler`, origin `pitchatlas://`)
   serving the island `dist` from `Resources/specimen/dist/` with correct MIME
   types for `.js/.wasm/.glb/.woff2/.webp`. (Pattern reference: a standard
   custom-scheme handler — a custom scheme is required because Vite/Three.js
   bundles break under `file://`.)
5. **`SpecimenWebView`** loads exactly one entry HTML per pitch
   (`pitchatlas://island/index.html#<slug>`), never navigates by router path.
6. **The dissolve:** a `scrollTransition` cross-fade from the WebView island to
   the native `SeamBall` (same seam math, so they can't disagree). `Haptics.lock()`
   fires once on the dissolve lock. Honor Reduce Motion (resolve straight to the
   native schematic).
7. **Verify on a real low-end device** in airplane mode on first launch: the 3D
   renders, drag-spins, and falls back to `SeamBall` if WebKit fails.
