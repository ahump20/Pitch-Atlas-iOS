# CLAUDE.md — Pitch Atlas iOS

Guidance for Claude Code working in this repository.

## What this is

Native SwiftUI iPhone app for **Pitch Atlas** — a sourced, interactive reference for how pitches are gripped and thrown. **Independent, standalone product.** It has no relationship to any other brand or product; no outside brand name, mark, identity, or account appears anywhere in the app or this repo. The web companion is `pitch-atlas.com` (`ahump20/Pitch-Atlas`).

- iOS 17.0 deployment floor. Swift (language version pinned in `project.yml`).
- Bundle ID: `com.pitchatlas.app`. Signing team is configured locally / in CI, not committed.
- The project file is **generated by XcodeGen** from `project.yml` — never hand-edit `PitchAtlas.xcodeproj` (it's gitignored). Change `project.yml`, run `xcodegen generate`.

## Non-negotiable doctrine (ported from the web repo)

- **Sourced, not corrected.** Every claim wears its source and confidence tier. A measured Statcast figure and a beat-writer quote never share a tier color. An unverified claim shows *as* unverified, with its required note — never hidden, never upgraded.
- **No mock data, no fabrication.** No `Math.random`, no placeholder arrays, no invented spin/velo. The grip photos carry "not tracked data." If a real source can't be determined, stop and ask.
- **No fabricated freshness.** Every "as of" computes from a real `retrievedAt`. No hardcoded "Updated today"/"Live".
- **Four states on every data surface.** loading (source-tagged skeleton) / error (a reason — the seal + alt-text tile, never a blank box) / empty (a reason) / populated (data + source + tier dot).
- **Rights-first images.** The visual actor is the seam specimen or first-party grip photography. No agency/player photos, no team/league marks, no re-hosted third-party footage.
- **Done = visitor-verified.** "Build succeeded" is not proof. Run it on the simulator/device and confirm what a user sees. For the offline promise, test in airplane mode on first launch.

## Content source of truth

The content lives in the **web repo** `~/code/Pitch-Atlas/src/data/` and is generated into `PitchAtlas/Resources/Content/*.json` by `tools/generate-content`. Never hand-author content JSON in this repo. The generator's drift guard reads expected record counts **from source at generate time** — never hardcode counts.

## Architecture

Native SwiftUI for the v1 binary: tab shell, searchable index, grip library, craftsmen hall, sources browser, and `SeamBall` specimens. State via `@Observable` + `@Environment` DI (no `ObservableObject`/Combine). Navigation: 5-tab `TabView` + per-tab `NavigationStack` + a `DeepLinkInbox` for `pitchatlas://` routes. The offline Three.js/WebView island is a v1.1 plan only; it is not part of the first submitted binary.

## v1 scope (locked)

Reference manual only — **no login, no Supabase client, no accounts, no community in the binary.** This sidesteps Sign in with Apple (4.8), all UGC moderation (1.2), and in-app account deletion (5.1.1(v)) for the first review. Community/Field-Notes is an additive v2 on the already-live Supabase backend. Privacy label = "Data Not Collected". Free. iPhone only. See `docs/COMPLIANCE.md`.

## Build / release

```bash
xcodegen generate
./scripts/build.sh            # iPhone simulator build
./scripts/build.sh test       # build + test
```

CI is **Xcode Cloud** (configured in the Xcode project). Three release rules: build numbers are monotonic per app and bump in lockstep across configs; an `action_required` result means a compile error, not an approval gate; release branches merge back to `main` the same day they archive. The post-clone step generates the Xcode project. Content refresh is a local pre-release step that must be committed before archive.

## Patterns

Authored for Pitch Atlas: native SwiftUI token-enum theming (`Color(hex:)` + `Font.custom(relativeTo:)` + system fallbacks), native seam specimens, CoreMotion foil treatment, a `PrivacyInfo.xcprivacy` manifest, and Xcode Cloud CI.
